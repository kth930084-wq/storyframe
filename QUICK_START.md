# 빠른 시작 가이드

## 🚀 새로운 기능들

### 1️⃣ 슬라이드 뷰 (PPT처럼)
**위치**: 에디터 페이지 → 사이드바 "슬라이드 뷰" 버튼

```
┌─────────────────────────────────────┐
│ 슬라이드 뷰                          │
├─────────────────────────────────────┤
│ 씬 1: 인트로           ↕ 드래그    │
│ 씬 2: 메인 장면        ↕ 가능      │
│ 씬 3: 아웃로           [×] 삭제   │
├─────────────────────────────────────┤
│ 총 3개 씬 / 30초                    │
└─────────────────────────────────────┘
```

**기능**:
- 씬 드래그로 순서 변경
- 클릭으로 씬 선택
- [×] 버튼으로 삭제
- 통계 자동 계산

---

### 2️⃣ 빈 페이지 추가
**위치**: 사이드바 "빈 페이지" 버튼

**4가지 종류**:

#### A. 씬 구분 (Scene Divider)
- 📋 장면 사이의 제목 카드
- 큰 텍스트로 "제 2부" 같은 제목 표시

#### B. 자유 메모 (Free Memo)
- 📝 자유로운 텍스트 노트
- 마크다운 형식 지원

#### C. 직접 그리기 (Sketch Canvas)
- 🎨 이미지 업로드 공간
- 드래그 앤 드롭 지원

#### D. 이미지 업로드 (Image Upload)
- 🖼️ 이미지 파일 전용
- 클릭해서 선택하거나 드래그 가능

**사용 방법**:
```
1. "빈 페이지" 버튼 클릭
2. 원하는 유형 선택
3. 자동으로 새 씬 생성 + 편집 화면 표시
```

### 3️⃣ 캔버스 에디터 (향후 사용)
준비된 기능 (아직 UI 통합 필요):

각 요소는:

- 색상 변경 가능

## 🐛 문제 해결

### 슬라이드 뷰가 안 보임
```
확인할 것:
1. slideViewMode 상태 확인 (true/false)
2. SlideView 컴포넌트 임포트 확인
3. 브라우저 콘솔에 에러가 있는지 확인
```

### 빈 페이지가 편집 안 됨
```
확인할 것:
1. blank_page_type이 설정되었는지 확인
2. BlankPageContent 컴포넌트가 렌더링되는지 확인
3. handleUpdateScene 콜백이 작동하는지 확인
```

### 데이터가 저장 안 됨
```
확인할 것:
1. localStorage 설정 확인
2. addToHistory 호출 확인
3. handleUpdateScene 호출 확인
```

## 💡 팁

1. **슬라이드 뷰는 읽기 전용 아님**
   - 썸네일을 클릭하면 메인 에디터에서 수정 가능
   - 드래그로 순서 변경 가능

2. **빈 페이지는 일반 씬처럼 작동**
   - 이미지나 기존 필드 사용 안 함
   - blank_page_content에만 저장됨

3. **데이터 로드**
   - 빈 페이지 필드가 없는 기존 프로젝트도 OK
   - 자동으로 기본값 설정됨
